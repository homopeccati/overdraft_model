# ar8 · Plain-language audit first `[process]`

Before writing any refactored code, produce a plain-language audit of the target: list every problem you see, ranked by severity. Wait for confirmation before proceeding. Do not mix analysis and implementation in the same response.

**Where to use:** Unfamiliar or complex codebases, before any major refactor.
**Risk if omitted:** Skipping the audit means the LLM fixes the wrong things with confidence.
